# 42장 비동기 프로그래밍

## 42-1. 동기처리와 비동기처리

실행 컨텍스트 챕터에서 살펴본 것처럼, 1) 함수를 호출하면 2) 함수 코드가 평가되고 3) 함수 실행 컨텍스트가 생성된다.

이때 함수 실행 컨텍스트는 실행 컨텍스트 스택(콜스택)에 푸시되고 함수 코드가 실행되며, 종료 시 팝 되어 제거된다.

```javascript
const foo = () => {};
const bar = () => {};
foo();
bar();
```

![image.png](/images/ch42/img3.png)

### 자바스크립트 엔진의 특징

- 자바스크립트 엔진은 **단 하나의 실행 컨텍스트 스택**을 갖는다
- 함수를 실행할 수 있는 곳이 하나다 = 동시에 2개 이상의 함수를 실행할 수 없다
- 실행 컨텍스트 스택의 최상위 요소인 "실행 중인 실행 컨텍스트"를 제외한 모든 요소들은 실행 대기중인 task들이다
- 따라서 현재 실행 중인 실행 컨텍스트가 팝되어야지 실행이 가능하다

### 동기 처리 방식

자바스크립트 엔진은 한 번에 하나의 태스크만 실행할 수 있는 **싱글 스레드 방식**으로 동작하며 **블로킹**(blocking, 작업중단)이 발생한다. 이 방식을 동기 처리 방식이라고 한다.

![image.png](/images/ch42/img2.png)

### 비동기 처리 방식

`setTimeout` 함수는 일정 시간이 경과한 이후에 콜백 함수를 호출하지만 이후의 함수를 블로킹 하지 않고 곧바로 실행한다. 이처럼 현재 태스크가 종료되지 않아도 곧바로 실행하는 방식을 비동기 처리라고 한다.

![image.png](/images/ch42/img1.png)

### 동기처리 vs 비동기처리 비교

| 구분 | 동기처리 | 비동기처리 |
|------|----------|------------|
| 실행 순서 보장 여부 | 실행 순서 보장 | 실행 순서 보장 안됨 |
| 블로킹 발생여부 | 블로킹 발생 | 블로킹 발생 안함 |

JS의 타이머 함수인 `setTimeout`, `setInterval`, HTTP 요청, 이벤트 핸들러는 비동기처리방식으로 동작한다.

## 42-2. 이벤트루프와 태스크 큐

자바스크립트의 특징 중 하나는 싱글 스레드로 동작한다는 것. 이 싱글 스레드에서 동시성을 지원하는 방식이 **이벤트 루프**이다. 이건 브라우저에 내장되어 있는 기능 중 하나이다.

### 자바스크립트 엔진의 구조

대부분의 자바스크립트 엔진은 크게 2개의 영역으로 구분할 수 있다:

**1. 콜스택 (Call Stack)**
- 소스코드 평가 과정에서 생성된 **실행 컨텍스트**가 추가되고 제거되는 스택 자료구조
- 함수를 호출하면 함수 실행 컨텍스트가 순차적으로 콜 스택에 푸시되어 실행됨
- 자바스크립트 엔진은 **단 하나의 콜 스택**을 사용하므로, 현재 실행 중인 실행 컨텍스트가 종료되어 제거되기 전까지는 다른 어떤 태스크도 실행되지 않음

**2. 힙 (Heap)**
- **객체가 저장되는 메모리 공간**
- 콜 스택의 요소인 실행 컨텍스트는 힙에 저장된 객체를 참조
- 객체는 원시 값과 달리 크기가 정해져 있지 않으므로, 할당해야 할 메모리 공간의 크기를 런타임에 결정(**동적 할당**)해야 함
- 따라서 힙은 구조화되어 있지 않다는 특징이 있음

### 브라우저/Node.js의 역할

자바스크립트 엔진은 단순히 태스크가 요청되면 콜 스택을 통해 요청된 작업을 순차적으로 실행할 뿐이다. 

비동기처리에서 소스코드의 평가와 실행을 제외한 모든 처리는 자바스크립트 엔진을 구동하는 환경인 **브라우저 또는 Node.js가 담당**한다.

브라우저 또는 Node.js은 이를 위해 태스크 큐와 이벤트 루프를 제공한다.

![image.png](/images/ch42/img4.png)

**태스크 큐 (task queue/event queue/callback queue)**
- `setTimeout`이나 `setInterval`과 같은 비동기 함수의 콜백 함수 또는 이벤트 핸들러가 일시적으로 보관되는 영역
- 태스크 큐와는 별도로 프로미스의 후속 처리 메서드의 콜백 함수가 일시적으로 보관되는 **마이크로태스크 큐**도 존재

**이벤트 루프 (event loop)**
- 콜 스택에 현재 실행 중인 실행 컨텍스트가 있는지, 그리고 태스크 큐에 대기 중인 함수가 있는지 반복해서 확인
- 만약 콜 스택이 비어 있고 태스크 큐에 대기 중인 함수가 있다면, 이벤트 루프는 순차적(FIFO)으로 태스크 큐에 대기 중인 함수를 콜 스택으로 이동시킴
- 이때 콜 스택으로 이동한 함수는 실행됨
- 즉, 태스크 큐에 일시 보관된 함수들은 비동기 처리 방식으로 동작
